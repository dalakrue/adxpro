# Evidence, Assumptions, and Limitations — 2026-06-20

## Evidence available in the package

- completed-H1 cutoff and appended-future invariance tests;
- chronological settled prediction schema support;
- deterministic module outputs for the same canonical identity/input hash;
- synthetic no-shift/shift, anomaly injection, label-prior shift, subgroup calibration, CRC monotonicity, RevIN round-trip, DML placebo/treatment, IRM, Group DRO, and signature-bound tests;
- canonical ID/generation/timestamp synchronization tests;
- full existing regression suite;
- Streamlit health startup;
- bounded runtime/memory benchmark.

## Unavailable or not validly calculable from the uploaded evidence

1. **Production accuracy uplift:** no sufficient new out-of-sample settled chronological live evidence exists after this implementation.
2. **Formal CRC guarantee for EURUSD H1:** serial dependence and changing regimes violate a simple iid interpretation; the output is an operational chronological bound with an explicit assumption note.
3. **RevIN superiority and enabled influence:** the existing forecast contributors do not expose a complete compatible normalized-input training/inference interface plus sufficient purged settled comparison columns. Original and round-trip RevIN points are retained; influence is disabled.
4. **TP-first/SL-first BBSE correction:** the settled schema lacks a validated pre-outcome touch-class predictor. Observed touch base rates can be reported, but no causal/predictive correction is applied.
5. **Live DML event effects:** there is no aligned, validated historical event-treatment indicator in the supplied application data. The offline implementation and synthetic tests are present, but no EUR/USD causal effect is fabricated.
6. **Exact dynamic RRCF co-displacement:** the bundled dependency-free implementation is a bounded deterministic random-cut-tree forest approximation. Exact paper-equivalent codisp is not claimed.
7. **Production IRM/Group-DRO superiority:** these are diagnostics/validation controls; they do not replace weights without separate settled walk-forward superiority.
8. **Phone heat/battery improvement:** no physical mobile measurement was performed.
9. **Live API timing/synchronization anomaly calibration:** fields are supported, but no fresh credentialed live run was available to estimate production distributions.
10. **Causal identification:** DML still depends on treatment timing, overlap, no unmeasured confounding, and valid pre-treatment covariates.

## Safe insufficient-evidence behavior

When evidence is absent or assumptions fail, the existing protected decision is preserved, research confidence is lowered, adjustment is skipped, and the reason is published. The system does not invent priors, effects, calibration guarantees, or anomaly certainty.
